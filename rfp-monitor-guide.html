★ Top 5 Search Tips

1
Search on Tuesday or Wednesday morning. Government agencies post new solicitations Monday–Tuesday. Searching mid-week catches everything before deadlines close — and leaves you time to act on what you find.
2
Use 3–5 specific words, not one. "Health communications RFP Maryland" returns far better results than "communications" alone. The more context you give, the more targeted the results.
3
Rotate keywords every session. "Public health outreach," "behavior change consulting," and "community engagement RFP" each surface different opportunities. Never rely on the same search twice in a row.
4
Keep 5–8 sources enabled per search. Too many sources at once can hit your API rate limit. Toggle sources in the Sources tab — green means included in the next search.
5
Rate limit error? Wait 60 seconds. It resets automatically. Each search costs approximately 5¢ — your $5 in API credits covers roughly 100 searches at weekly use.

★ Best Day & Time to Search

Tuesday or Wednesday morning is the ideal time for your weekly search session.

Most federal and government procurement offices post new solicitations at the start of the work week. Foundations and nonprofits tend to post mid-week. Searching Tuesday or Wednesday morning catches everything posted since the previous week — with enough time left in the week to respond to anything promising.

Why not Monday or Friday?

  • Avoid Mondays — postings from that morning won't be indexed by search engines yet
  • Avoid Fridays — deadlines frequently fall on Fridays, leaving no time to act on new findings

Recommended weekly rhythm

DayAction
Tue / Wed morningRun 2–3 searches with different keyword combinations. Review results and add to pipeline.
ThursdayCheck manual portals (login-required sites). Update statuses on existing pipeline opportunities.
FridayRun the weekly digest. Note approaching deadlines for the following week.

1 Overview

The RFP Monitor searches multiple procurement, grant, and communications-specific sources simultaneously using the Anthropic API with Claude's web search capability. Results are returned for your review before being added to a full opportunity pipeline — from discovery through award.

What it does

  • Searches up to 23 auto-searchable sources in one click
  • Returns matched opportunities for review before adding to your pipeline
  • Tracks opportunities through: New → Pursuing → Submitted → Won / Lost
  • Provides a Go/No-Go scorecard for each opportunity
  • Links to manual portals requiring login for weekly review
  • Generates a weekly digest summary of your pipeline

How it connects

Auto-searchAnthropic API (claude-sonnet-4-5) with web search tool
Proxypop-proxy.netlify.app — routes API calls through Netlify to bypass browser security restrictions
Data storageBrowser localStorage — pipeline and sources save locally per device
Manual portalsDirect links to login-required procurement portals

2 Initial Setup

Getting your API Key

The RFP Monitor requires an Anthropic API key to run searches. You only need to enter this once per device.

  1. Go to console.anthropic.com and sign in
  2. Click API Keys in the left sidebar
  3. Click Create Key — name it "RFP Monitor"
  4. Copy the full key immediately — it starts with sk-ant- and is only shown once
  5. Open the RFP Monitor and paste the key into the API key field
  6. Click Save key
Important: The key is only shown once when created. If you navigate away before copying it, you will need to delete it and create a new one. Always copy it immediately.

API Credits & Cost

Add prepaid credits at console.anthropic.com → Billing. Credits are prepaid — you cannot be charged more than you load.

3 Running a Search

Search Tab fields

FieldWhat to enter
Focus keywordsYour search topic — 3–5 descriptive words
Limit to org typeFilter to Government, Foundation, Nonprofit, or Academic sources only
Exclude keywordsTopics to skip — e.g. "clinical trials, pharmaceutical"
Geographic focusOptional location — e.g. "Maryland, DC, federal"

The counter next to the search button shows how many sources are currently enabled. Click Search all sources → to run.

Reviewing Results

Results appear in a review panel after the search completes (typically 20–40 seconds). For each result you can:

  • Add → — adds it to your pipeline with status "New"
  • Skip — dismisses it without adding
  • Add All — adds every result to your pipeline at once
Results are not saved automatically. Review them before navigating away or running another search — a new search replaces the current results panel.

Enabling & Disabling Sources

Each source card in the Sources tab has an On/Off toggle button. Green = included in search, gray = excluded. Changes save automatically. Start with 5–8 sources enabled to stay within rate limits.

Keep SAM.gov and Philanthropy News Digest enabled by default — they are the highest-volume sources for your niche.

Manual Portals

Manual portals are sites requiring a login or CAPTCHA — they cannot be searched automatically. The Manual Portals tab provides direct links so you can check them quickly during your Thursday review session.

Go/No-Go Scorecard

Click the scorecard button on any pipeline opportunity to run a quick evaluation. Rate six criteria: mission fit, timeline, budget, competitive position, existing relationship, and capacity. The tool calculates a score and recommended decision. This connects with your Go/No-Go form in the full Operations Hub.

6 Weekly Digest

The Digest tab generates a plain-language summary of your current pipeline — how many opportunities are active, what's coming up, and key deadlines. Run it every Friday as part of your weekly rhythm.

  • Click Generate digest to create the summary
  • The digest reflects your pipeline at the moment you generate it
  • Copy and paste into an email or document as needed

7 Troubleshooting

ErrorFix
Failed to search / CORS errorCheck that pop-proxy.netlify.app is live. Verify your key at console.anthropic.com.
Invalid x-api-keyRe-enter your API key. Create a new one at console.anthropic.com if needed — keys are only shown once.
Rate limit errorWait 60 seconds and retry. Reduce the number of enabled sources before searching.
Could not parse resultsTry more specific keywords and search again.
No results foundBroaden or change keywords. Confirm sources are enabled in the Sources tab.
Pipeline disappearedBrowser localStorage was cleared. Data is device-local — this is a known limitation.
