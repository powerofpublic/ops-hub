★ Top 5 Search Tips

1
Search on Tuesday or Wednesday morning. Government agencies post new solicitations Monday–Tuesday. Searching mid-week catches everything before deadlines close — and leaves you time to act on what you find.
2
Use 3–5 specific words, not one. "Health communications RFP Maryland" returns far better results than "communications" alone. The more context you give, the more targeted the results.
3
Rotate keywords every session. "Public health outreach," "behavior change consulting," and "community engagement RFP" each surface different opportunities. Never rely on the same search twice in a row.
4
Keep 5–8 sources enabled per search. Each source card has an On/Off toggle — green means included in the next search, gray means excluded. Too many at once can hit your API rate limit.
5
Rate limit error? Wait 60 seconds, then click ⏱ 60s cooldown in the header to track the reset. Each search costs approximately 5¢ — your $5 covers roughly 100 searches at weekly use.

★ Best Day & Time to Search

Tuesday or Wednesday morning is the ideal time for your weekly search session.

Most federal and government procurement offices post new solicitations at the start of the work week. Foundations and nonprofits tend to post mid-week. Searching Tuesday or Wednesday morning catches everything posted since the previous week — with enough time left in the week to respond to anything promising.

Why not Monday or Friday?

  • Avoid Mondays — postings from that morning won't be indexed by search engines yet
  • Avoid Fridays — deadlines frequently fall on Fridays, leaving no time to act on new findings

Recommended weekly rhythm

1 Overview

The RFP Monitor searches multiple procurement, grant, and communications-specific sources simultaneously using the Anthropic API with Claude's web search capability. Results are returned for your review before being added to a full opportunity pipeline — from discovery through award.

What it does

  • Searches up to 23 auto-searchable sources in one click
  • Returns matched opportunities for review before adding to your pipeline
  • Tracks opportunities through: New → Pursuing → Submitted → Won / Lost
  • Provides a Go/No-Go scorecard for each opportunity
  • Links to manual portals requiring login for weekly review
  • Generates a weekly digest summary of your pipeline

How it connects

Auto-searchAnthropic API (claude-sonnet-4-5) with web search tool
Proxypop-proxy.netlify.app — routes API calls through Netlify to bypass browser security restrictions
Data storageBrowser localStorage — pipeline and sources save locally per device
Manual portalsDirect links to login-required procurement portals

2 Initial Setup

Getting your API Key

The RFP Monitor requires an Anthropic API key to run searches. You only need to enter this once per device.

  1. Go to console.anthropic.com and sign in
  2. Click API Keys in the left sidebar
  3. Click Create Key — name it "RFP Monitor"
  4. Copy the full key immediately — it starts with sk-ant- and is only shown once
  5. Open the RFP Monitor and paste the key into the API key field
  6. Click Save key
Important: The key is only shown once when created. If you navigate away before copying it, you will need to delete it and create a new one. Always copy it immediately.

3 Running a Search

How the Search tab works

The Search tab provides five copy-ready prompts you use directly in your Claude Pro subscription at claude.ai — at no extra cost. Claude searches the live web and returns real, current RFPs with direct links and descriptions.

PromptWhat it searches
Federal & governmentSAM.gov and Grants.gov — active RFPs, last 14 days
Foundations & nonprofitsPhilanthropy News Digest, CDC Foundation, Grantmakers in Health, NACCHO, NNPHI
Communications-specificO'Dwyer's PR, Everything PR, rfpdb.com
Regional — Maryland & DCeMMA, DC OCP, Virginia EPROCURE, HigherGov
Broad weekly sweepAll sources, last 14 days, minimum 8 results

Workflow — Search to Pipeline

  1. Go to the Search tab and click Copy prompt
  2. Open claude.ai in a new tab and paste the prompt into a new chat
  3. Claude searches the live web and returns results with titles, deadlines, descriptions, and direct links
  4. Copy the full Claude response
  5. Return to your portal → Pipeline tab → paste into Import from Claude
  6. Click + Add all to pipeline — everything imports in one click
Run 2–3 different prompts per session for the best coverage. Each prompt targets different source types so results don't overlap.
Best day: Tuesday or Wednesday morning. Government agencies post Monday–Tuesday so mid-week searches catch everything.

4 Managing Sources

Sources — now in the Portals tab

All 23 sources are now organized as direct links in the Portals tab — divided into Government & procurement, Foundations, Nonprofit & associations, and Academic sections alongside the login-required manual portals. Sources are permanently stored in the HTML file and never in your browser.

Enabling & Disabling Sources

Each source card in the Sources tab has an On/Off toggle button. Green = included in next search, gray = excluded. Your toggle settings save automatically to your browser. Start with 5–8 sources enabled to stay within rate limits.

Keep SAM.gov and Philanthropy News Digest enabled by default — they are the highest-volume sources for your niche.

Adding or Removing Sources

The Sources tab has a single combined info box — "About Auto-Searchable Sources" — that explains both what the sources are and how to manage them. There is no Add Source button in the UI. This is intentional.

Why there's no Add Source UI: Sources added through a browser UI save to localStorage and disappear when you clear browser data or switch devices. Storing them permanently in the HTML file is safer and more reliable.

To add or remove a source:

  1. Message Claude: "Add [source name], URL: [url], Type: Government / Foundation / Nonprofit / Academic to the RFP Monitor"
  2. Download the updated rfp-monitor.html
  3. Upload to GitHub — the source is live everywhere instantly
This two-minute process permanently bakes the source into the HTML so it survives any future update, device switch, or browser clear.

Manual Portals

Manual portals are sites requiring a login or CAPTCHA — they cannot be searched automatically. The Manual Portals tab provides direct links so you can check them quickly during your Thursday review session.

Like auto-searchable sources, portals are permanently stored in the HTML file — not your browser. There are no Add buttons in the UI. To add or remove a portal, ask Claude: share the name, URL, and which section it belongs to. Upload the updated file to GitHub and it is live everywhere instantly.

★ Rate Limit & Cooldown Timer

The Anthropic API limits you to 30,000 input tokens per minute. Each search uses approximately 8,000–15,000 tokens depending on how many sources are enabled. Searching too quickly back-to-back will trigger a rate limit error.

Wait 60 seconds between searches. The rate limit resets every minute automatically.

Manual Cooldown Timer

A ⏱ 60s cooldown button lives permanently in the header bar (top right, next to ← Hub). After your search results appear, click it to start a 60-second countdown so you know exactly when it is safe to search again.

  • Counter turns amber when under 15 seconds remaining
  • Shows ✓ green when ready — resets automatically after 3 seconds
  • Click again mid-countdown to cancel it
  • Completely separate from search logic — clicking it never triggers or affects a search
The timer is manual by design. You start it after results appear, so it counts down while you review results — not while waiting for the search to complete.

"Failed to fetch" errors

"Failed to fetch" is a network error — it means the browser could not reach the Netlify proxy. The search never ran and no tokens were used. It is not a rate limit issue. Common causes:

  • Proxy went cold — free Netlify functions occasionally need a moment to wake up. Try again after 10–15 seconds.
  • Brief internet drop — a momentary connection loss during the 20–40 second search window
  • Proxy not deployed — check app.netlify.com → pop-proxy → Functions to confirm proxy is listed

Go/No-Go Scorecard

Click the scorecard button on any pipeline opportunity to run a quick evaluation. Rate six criteria: mission fit, timeline, budget, competitive position, existing relationship, and capacity. The tool calculates a score and recommended decision. This connects with your Go/No-Go form in the full Operations Hub.

6 Weekly Digest

The Digest tab generates a plain-language summary of your current pipeline — how many opportunities are active, what's coming up, and key deadlines. Run it every Friday as part of your weekly rhythm.

  • Click Generate digest to create the summary
  • The digest reflects your pipeline at the moment you generate it
  • Copy and paste into an email or document as needed
